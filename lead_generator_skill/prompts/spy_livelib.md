# /lead spy-livelib — топ книг ниши с LiveLib

## Что делает

Собирает **топ книг по теме/нише** с LiveLib.ru:
- Название книги
- Автор
- Рейтинг (если доступен)
- Количество оценок
- Прямая ссылка на книгу

Используется для:
- 📚 **Блок «Что почитать»** в контент-материалах (статьи, лид-магниты, рассылки)
- 🎯 **SEO-статьи** вида «Топ-10 книг по теме X» (перелинковка с основной статьёй)
- 💡 **Идеи для постов** в соцсетях (мини-обзоры книг ниши)
- 🧠 **Карта интересов ЦА** — что читает аудитория, какие тренды в саморазвитии/психологии

## Команды

```bash
# Топ-15 книг по теме «когнитивные искажения»
/lead spy-livelib "когнитивные искажения"

# Топ-20 с минимальным рейтингом 4.0 (только стоящие)
/lead spy-livelib "психология отношений" --min-rating 4.0

# Под ручную (прямой запуск скрипта)
PYTHONIOENCODING=utf-8 python scripts/spy_livelib_books.py \
  --query "медитация для начинающих" --limit 20 --min-rating 4.0
```

## Выход

- **Консоль:** красивая таблица с топ-3 по рейтингу
- **JSON (опционально):** `livelib_books_<запрос>.json` с полной структурой
- **CSV (опционально):** для импорта в Google Sheets

## Алгоритм

1. Playwright открывает `livelib.ru` (получает DDoS-Guard cookies)
2. Заполняет форму поиска `input[name="filter[search]"]` + Enter
3. Парсит AJAX-выдачу: `brow-title` ссылки + авторов + рейтинги
4. Дедуп по `book_id`, фильтр по `min-rating` (если задан)
5. Сохраняет в JSON/CSV

## Технические нюансы

- **DDoS-Guard обход** — через Playwright с прогревом главной (cookies + challenge)
- **GET /find?q= редиректит** на главную — поэтому submit формы вместо GET
- **Рейтинги подтягиваются не всегда** — порядок `data-rating` блоков в HTML не совпадает с порядком карточек на 13.06.2026 (AJAX-выдача). Title/author/url — основное, что надёжно
- **Headless=True по умолчанию** — для отладки добавь `--debug` (браузер будет видим + сохранит HTML)

## Связь с expert-reviews-hub

**НЕ дублирует, а дополняет:**
- `spy_livelib_books.py` (этот скил) — **ТОП книг ниши** для контента
- `expert-reviews-hub/scripts/parse_livelib.py` — **ОТЗЫВЫ на конкретную книгу** для соцдока

```
/lead spy-livelib "когнитивные искажения"   →  список книг для блока "что почитать"
   ↓
expert-reviews-hub parse_livelib <book_url>  →  цитаты и плюсы/минусы для лендинга
```

## Границы

- ❌ Не парсит рецензии (это `parse_livelib.py` из expert-reviews-hub)
- ❌ Не парсит «похожие книги» / «рекомендации»
- ❌ Не собирает обложки (только title + url)
- ⚠️ 1 страница выдачи = ~5-8 книг. Для 20+ — потребуется пагинация (в разработке)
