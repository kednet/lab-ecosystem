# /lead wordstat — авторизованный сбор семантики через Яндекс.Вордстат

## Что делает

Команда `/lead wordstat [черновик.md]` — берёт черновик фраз, ведёт пользователя через авторизацию в Яндексе, пошагово собирает частотности через Playwright (с авторизованной сессией), нормализует, кластеризует по сортам А/Б/В/Г/Д, выдаёт 4 файла + сводку.

**Это полный конвейер:** черновик фраз → точные цифры → сортировка → title/h1 рекомендации → контент-план.

## Когда использовать

- **При запуске нового проекта/сайта** — собрать семантическое ядро
- **Перед SEO-аудитом** — получить реальные цифры для обоснования рекомендаций
- **Каждые 2-3 месяца** — отслеживать динамику спроса (что растёт, что падает)
- **При выборе тем для контента** — что реально ищут, а не «что мне кажется»
- **При планировании рекламных кампаний** — коммерческие запросы для Я.Директа

## Вход

- **Черновик-файл** (markdown) со списком 20-30 фраз — обязательно
  - Шаблон: `templates/keyword_draft.md`
  - Готовый пример: `data/draft_lab_zhelanij_keywords.md`
- **Авторизованная сессия в Яндексе** (через Playwright) — обязательно
  - Пользователь логинится в браузере Playwright один раз
  - Сессия сохраняется в cookies, повторные запуски без логина
- **Конфиг проекта** (опционально) — `data/project_<slug>.json`
  - Если задан — скил генерирует рекомендации по title/h1 сразу по URL-ам
  - Если не задан — выдаёт общие рекомендации без привязки

## Выход (4 файла)

1. **`data/wordstat_<slug>_<date>.md`** — таблица частотностей (формат 02-wordstat-raw.md)
2. **`data/wordstat_<slug>_clusters_<date>.md`** — сортировка А/Б/В/Г/Д + наблюдения
3. **`data/wordstat_<slug>_titles_<date>.md`** — рекомендации по title/h1/description
4. **`data/wordstat_<slug>_plan_<date>.md`** — контент-план на месяц

Где:
- `<slug>` — kebab-case название проекта (например `lab-zhelanij`)
- `<date>` — дата сбора в формате `YYYY-MM-DD`

## Алгоритм работы скила (5 шагов)

### Шаг 1: Прочитать черновик

```
Read <путь к черновику>
```

Валидация:
- Файл существует и не пустой
- Содержит секцию `## ТОП-N для парсинга` (иначе ошибка)
- 20 ≤ N ≤ 50 (если больше — предупредить, если меньше — спросить)

### Шаг 2: Спросить про авторизацию

```
Скил: "Открой wordstat.yandex.ru, залогинься в Яндекс ID,
       скажи готово. Логин не сохраняется — сессия живёт в браузере Playwright."
```

### Шаг 3: Запустить Python-скрипт

```bash
python scripts/wordstat_playwright.py \
  --input <черновик> \
  --output data/wordstat-<slug>-<date>/ \
  --rate-limit 5
```

Скрипт работает 3-5 минут (25 фраз × 5-7 сек + обработка).
Сохраняет JSON в `data/raw/wordstat_<slug>_<timestamp>.json`.

### Шаг 4: Сгенерировать 4 файла

На основе JSON из шага 3 скил создаёт 4 markdown-файла по шаблонам ниже.

### Шаг 5: Вернуть сводку

Финальный ответ в чате: топ-10 фраз + 3 инсайта + ссылки на файлы + рекомендации.

## Сортировка А/Б/В/Г/Д (по частотности)

| Группа | Диапазон | Действие |
|--------|----------|----------|
| 🔥 **А (горячие)** | 1000+ | Отдельные целевые страницы |
| 🟢 **Б (тёплые)** | 500–9999 | В title/description/h1 |
| 🟡 **В (средние)** | 200–499 | В h2 и alt |
| 🟠 **Г (холодные)** | 10–199 | В тексте |
| 🟤 **Д (наш продукт)** | <10 | Брендовые, в nav/footer |

```bash

  → генерит 4 файла
```

## Edge cases

- **Капча в Вордстате** → скрипт паузит 30 сек, повторяет. Если не помогло — флаг в JSON, скил продолжает с тем что есть
- **Нет авторизации** → скрипт ловит редирект на логин, скил просит авторизоваться
- **Пустой результат (frequency = 0)** → записать с флагом `no_data: true`, не падать
- **Файл не в формате черновика** → скил просит переделать
- **N < 20** → скил спрашивает, точно ли парсить так мало
- **N > 50** → скил предупреждает про скорость (сбор ~10 минут)

## Границы (что НЕ делаем)

- ❌ Парсинг Google Trends, Yandex.Trends (только Wordstat)
- ❌ Парсинг конкурентов (это команды `/lead spy`, `/lead spy-trends`)
- ❌ Генерация статей «на лету» (это команды `/lead dzen`, `/lead seo-funnel`)
- ❌ UI для редактирования черновика (только markdown-файлы)
- ❌ Интеграция с Метрикой/Вебмастером (отдельная задача)
