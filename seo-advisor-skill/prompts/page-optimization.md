# Оптимизация страницы (главный промпт)

## Назначение
Полная SEO-оптимизация одной страницы: title/description/og/Schema/H1-H3/LSI/E-E-A-T/технические факторы.

## Вход
- URL (WebFetch) ИЛИ
- HTML/текст страницы ИЛИ
- Описание страницы (если страница ещё не создана — «спроектируй SEO для будущей страницы о X»)

## Алгоритм

### Шаг 0. Определи интент (см. SKILL.md «Шаг 0»)
Не пропускай. Без интента нельзя понять, что должно быть в title.

### Шаг 1. Собери текущее состояние
Через WebFetch → HTML → grep:
- `<title>...</title>`
- `<meta name="description">`
- `<meta name="keywords">`
- `<meta property="og:...">`
- `<link rel="canonical">`
- `<h1>...</h1>`, `<h2>...`, `<h3>...`
- Schema JSON-LD блоки
- Текст: длина, повторы ключей, наличие даты, автора

Если WebFetch не сработал (404/таймаут/MITM):
- Попроси HTML напрямую: «Дай HTML страницы в блок кода»
- Или: «Опиши текст страницы 2-3 абзацами + основные заголовки»

### Шаг 2. Анализ по 8 блокам
Все блоки обязательны к заполнению.

#### Блок 1. Title
- **Сейчас:** + длина + ключ в начале?
- **Проверь:** 50-70 символов, ключ слева, бренд справа, уникальность
- **Рекомендация:** новая формулировка + обоснование
- **5 формул** (для ниши саморазвития):
  1. «{Название книги} — {Автор}: конспект, воркбук и отзывы»
  2. «{Тема}: {выгода} — блог Лаборатории желаний»
  3. «{Имя эксперта} — {специализация} | Лаборатория желаний»
  4. «{Тема}: Топ-{N} {что-то} в 2026 году»
  5. «{Действие} {объект}: {выгода} | {бренд}»

#### Блок 2. Meta Description
- **Сейчас:** + длина + наличие призыва?
- **Проверь:** 150-160 символов, начинается с глагола, уникально
- **Рекомендация:** новая формулировка
- **Формулы:**
  - Книга: «Читайте краткий конспект «{X}» {Автор}. Ключевые идеи, цитаты, практический воркбук. Подходит для саморазвития.»
  - Блог: «Узнайте {что}. {выгода}. Практические советы, примеры, пошаговый план.»
  - Лендинг: «{Имя} — {что делает}. Запишитесь на {услугу}: 30 минут, бесплатно.»

#### Блок 3. OG / VK / Twitter превью
- og:title (60-90 символов, не дублирует meta title полностью — допустима вариация)
- og:description (до 200 символов)
- og:image (1200×630, < 200 КБ, alt совпадает с title)
- og:url (canonical)
- og:type (article / website / book)
- og:site_name (Лаборатория желаний)
- og:locale (ru_RU)
- article:published_time
- article:modified_time
- article:author
- article:section
- twitter:card (summary_large_image)
- twitter:title / description / image
- vk:image (отдельный, может совпадать с og:image)

См. `templates/og-template.md` для готового блока.

#### Блок 4. Структура H1-H3
- H1 — ровно 1, совпадает с интентом
- H2 — 4-8 штук, иерархия без пропусков
- H3 — внутри H2, для подразделов
- Ключ/LSI обязательно в H1, желательно в H2

Стандартная структура для **страницы книги**:
```
H1: {Название} — полный конспект
H2: {Автор} — краткая биография
H2: О чём книга «{Название}»
H2: Ключевые идеи (5-7)
  H3: Идея 1
  H3: Идея 2
  ...
H2: Практические методы из книги
  H3: Метод 1
  H3: Метод 2
H2: Воркбук: задания по книге
H2: Цитаты
H2: Отзывы читателей
H2: Где купить и скачать
```

#### Блок 5. Ключевые слова + LSI
Не просто «плотность 2-3%» — современный SEO про **топикальное покрытие**.

**Главный ключ** (high intent): 1 штука
**Средние**: 2-3 штуки
**Длинный хвост**: 5-7 штук
**LSI-расширения** (см. `data/lsibase.md`): 10-15 слов/фраз, обязательные для темы
**PAA (People Also Ask)**: 5 вопросов (см. `prompts/faq.md`)

**Проверь в тексте:**
- Главный ключ в первых 100 словах ✅
- Ключ в H1 ✅
- LSI распределены по тексту естественно ⚠️
- Нет переоптимизации (плотность < 3%)

#### Блок 6. Schema.org (JSON-LD)
Какие блоки нужны — зависит от типа страницы:
- **Книга** → Book + BreadcrumbList + FAQPage (если есть FAQ) + Review (если есть отзывы)
- **Статья** → Article + BreadcrumbList + Person (автор) + Organization (издатель)
- **Эксперт** → Person + Organization + BreadcrumbList
- **Главная** → Organization + WebSite + BreadcrumbList
- **Лендинг** → Organization + Service/Product + FAQPage + BreadcrumbList

См. `prompts/schema.md` и `templates/schema/*.json` для готовых блоков.

#### Блок 7. E-E-A-T
См. `prompts/eeat.md` — детальный чек-лист. Краткий вид:
- **Experience** (опыт): кейсы, фото, скрины, даты
- **Expertise** (экспертиза): регалии, источники, методология
- **Authoritativeness** (авторитет): внешние ссылки, упоминания
- **Trustworthiness** (доверие): контакты, юр.лицо, SSL, политика, «О нас»

#### Блок 8. Техника
- alt у всех изображений (с ключом где уместно)
- canonical на саму себя
- sitemap.xml содержит страницу
- robots.txt не блокирует
- mobile-friendly (viewport, размер шрифта)
- Скорость: < 2 сек desktop, < 3 сек mobile
- Изображения webp/avif, lazy loading
- Schema валидна (запусти `scripts/schema-validate.py`)

### Шаг 3. Приоритеты
Отсортируй рекомендации по 3 уровням:
- 🔴 **Срочно** — без этого страница не ранжируется
- 🟡 **Важно** — заметный буст за 1-2 недели
- 🟢 **Желательно** — полировка

### Шаг 4. Чек-лист внедрения
Финальный чек-лист для разработчика:
- [ ] Изменить Title
- [ ] Добавить/изменить Description
- [ ] Добавить OG/VK/Twitter мета
- [ ] Добавить Schema JSON-LD
- [ ] Перестроить H1-H3
- [ ] Увеличить/переписать текст
- [ ] Добавить alt к изображениям
- [ ] Добавить внутренние ссылки (2-5)
- [ ] Проверить canonical/sitemap/robots
- [ ] Запустить schema-validate

## Пример вызова
```
/seo optimize https://lab.com/library/transerfing
/seo optimize
# (вставить HTML в блок кода)
/seo optimize
Тема: «Трансерфинг реальности» — страница книги ещё не создана, спроектируй SEO-структуру под будущую страницу
```
