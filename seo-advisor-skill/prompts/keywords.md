# Подбор ключевых слов (с LSI и PAA)

## Назначение
Генерация семантического ядра страницы: главный ключ, средние/НЧ, длинный хвост, LSI-слова, PAA-вопросы, интент-кластеризация.

## Вход
- Тема страницы (например, «исполнение желаний»)
- Или URL (тогда сначала WebFetch → текущие ключи → расширяем)

## Алгоритм

### Шаг 1. Определи seed-тему
Что ищет пользователь? Один корень смысла. Обычно это существительное + 1-2 уточнения.

Примеры:
- «конспект Трансерфинг реальности» → seed: трансерфинг реальности конспект
- «исполнение желаний» → seed: исполнение желаний
- «как бросить курить» → seed: бросить курить
- «психолог онлайн Москва» → seed: психолог онлайн москва

### Шаг 2. Сгенерируй 4 уровня частотности

#### Высокочастотные (ВЧ) — высокая конкуренция
- Главный «головной» запрос
- Обычно 1-2 слова, без уточнений
- **Пример для темы «исполнение желаний»:**
  - исполнение желаний (5000/мес, высокая К)
  - как исполнить желание (3000/мес, высокая К)
  - закон исполнения желаний (2500/мес, высокая К)

#### Среднечастотные (СЧ) — средняя конкуренция
- С уточнением («по методу X», «упражнения», «техника»)
- 2-4 слова
- **Пример:**
  - исполнение желаний по Зеланду (800/мес, средняя К)
  - воркбук исполнение желаний (500/мес, средняя К)
  - техники исполнения желаний (700/мес, средняя К)

#### Низкочастотные (НЧ) — низкая конкуренция
- Длинные, конкретные
- 4-8 слов
- **Пример:**
  - исполнение желаний по трансерфингу пошагово (50/мес)
  - 5 упражнений на исполнение желаний перед сном (40/мес)
  - воркбук исполнение желаний скачать pdf (30/мес)

#### Длинный хвост (long-tail) — вопросы пользователей
- Вопросительные, начинаются с «как», «что», «почему», «можно ли», «стоит ли»
- **Пример:**
  - можно ли исполнить желание силой мысли?
  - как быстро исполнить желание за 1 день?
  - что делать если желание не исполняется?
  - трансерфинг работает или это placebo?
  - какие книги про исполнение желаний лучшие?

### Шаг 3. LSI-слова (топикальное покрытие)
Современный поиск (BERT, MUM, Яндекс Палех) понимает **тему**, а не точные повторы. LSI-слова (Latent Semantic Indexing) — это термины, которые должны быть в тексте, чтобы страница считалась авторитетной по теме.

**Правила:**
- 10-20 LSI-слов на страницу
- Распределить по тексту естественно
- Не пытаться запихнуть все в один абзац

**Где брать LSI:**
1. Смотри `data/lsibase.md` (база по нишам)
2. Wikipedia по теме → первые 30 уникальных терминов
3. Related Searches в Яндексе/Google по главному ключу
4. Синонимы, гипонимы, смежные понятия
5. Автоподсказки поисковиков

**Пример для «исполнение желаний»:**
- визуализация
- аффирмация
- медитация
- карта желаний
- намерение
- подсознание
- энергия
- квантовое поле
- закон притяжения
- осознанность
- благодарность
- поток
- Вселенная
- сценарий жизни
- маятник
- слайды
- транс

### Шаг 4. PAA (People Also Ask) — блок FAQ
Google показывает блок «Другие также спрашивают» — это **готовый ключ под featured snippet**.

**Сгенерируй 5-10 вопросов:**
- Начни с «как», «что», «почему», «можно ли», «когда», «зачем», «что будет если»
- Каждый вопрос = отдельный H3 или блок FAQ
- Сразу под вопросом — короткий ответ 40-60 слов (для snippet)

**Пример:**
1. Как работает исполнение желаний по трансерфингу?
2. Сколько времени нужно, чтобы желание исполнилось?
3. Можно ли исполнить желание за 1 день?
4. Что делать, если желание не исполняется?
5. Какие техники исполнения желаний самые эффективные?

Подробнее — `prompts/faq.md`.

### Шаг 5. Интент-кластеризация
Сгруппируй все ключи по интенту. Это показывает, **сколько страниц** нужно для полного покрытия темы.

| Интент | Ключи (пример) | Нужна страница |
|--------|----------------|-----------------|
| **Информационный** | «исполнение желаний», «как работает трансерфинг» | hub-страница (главная категории) |
| **Коммерческий-исследование** | «лучшие книги по исполнению желаний», «рейтинг» | Топ-10 |
| **Транзакционный** | «купить книгу Трансерфинг», «скачать воркбук» | карточка книги |
| **Навигационный** | «Зеланд трансерфинг официальный» | карточка автора |
| **Сравнение** | «трансерфинг vs закон притяжения» | статья-сравнение |

**Правило:** 1 страница = 1 главный интент. Если у страницы смешано 2+ интента → **раздробить** на отдельные страницы.

### Шаг 6. Конкурентность и приоритизация
Оцени каждый ключ:
- **KD (Keyword Difficulty)** — низкая/средняя/высокая
- **Сезонность** — есть ли пики (Новый год, весна, осень)
- **Релевантность проекту** — соответствует ли теме «Лаборатории желаний»

**Матрица:**
| Тип ключа | KD | Релевантность | Приоритет |
|-----------|-----|---------------|-----------|
| НЧ длинный хвост | низкая | высокая | 🥇 быстрая победа |
| СЧ с уточнением | средняя | высокая | 🥈 среднесрочно |
| ВЧ головной | высокая | высокая | 🥉 долгосрочно, через hub |
| Любой | низкая/средняя | низкая | ⛔ пропустить |

## Выходной формат

```markdown
## Семантическое ядро: {тема}

### Главный ключ (1 шт.)
> {ключ} — {частота} — {KD} — {интент}

### ВЧ — высокая конкуренция
| Ключ | Частота/мес | KD | Интент |
|------|-------------|-----|--------|
| ... | ... | ... | ... |

### СЧ — средняя конкуренция
| Ключ | Частота/мес | KD | Интент |
|------|-------------|-----|--------|
| ... | ... | ... | ... |

### НЧ — низкая конкуренция (длинный хвост)
| Ключ | Частота/мес | KD | Интент |
|------|-------------|-----|--------|
| ... | ... | ... | ... |

### PAA-вопросы (для FAQ-блока)
1. {вопрос} → 40-60 слов ответ
2. ...
3. ...

### LSI-слова (топикальное покрытие)
{список 10-20 слов/фраз}

### Интент-кластеры
- {интент}: {N ключей} → нужна {тип страницы}
- ...

### Приоритетный план
1. 🥇 Быстрые победы: {НЧ с высокой релевантностью}
2. 🥈 Среднесрочно: {СЧ с уточнением бренда/продукта}
3. 🥉 Долгосрочно: {ВЧ через hub-spoke}
```
