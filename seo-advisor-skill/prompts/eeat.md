# E-E-A-T аудит (критично для YMYL)

## Что такое E-E-A-T
Google (с 2022) официально требует для YMYL-тем (Your Money or Your Life) четырёхфакторную проверку:

- **E**xperience — реальный опыт автора по теме
- **E**xpertise — профессиональная экспертиза
- **A**uthoritativeness — авторитет в глазах других экспертов/СМИ
- **T**rustworthiness — надёжность, прозрачность, безопасность

**Темы YMYL для «Лаборатории желаний» (ОБЯЗАТЕЛЬНО к E-E-A-T):**
- Саморазвитие
- Психология
- Психотерапия
- Коучинг
- Финансы через мышление
- Здоровье через образ мыслей
- Отношения, семья
- Карьера
- Духовные практики

**НЕ-YMYL:** развлечение, художественная литература, новости, лайфстайл-блогинг, нон-фикшн-разборы книг с дисклеймером «не заменяет консультацию специалиста».

## Зачем E-E-A-T в скиле
Без явных E-E-A-T-маркеров страницы по саморазвитию рискуют:
- Терять позиции в Google (особенно после Helpful Content Update 2024)
- Получать «недоверенный» ярлык в Quality Rater Guidelines
- Проигрывать конкурентам с тем же контентом, но лучшим E-E-A-T

## Алгоритм аудита

### Шаг 1. Определи, YMYL ли страница
Задай вопрос: «Может ли контент этой страницы причинить вред здоровью/финансам/психике читателя, если он ему поверит?»
- Да → YMYL → полный E-E-A-T обязателен
- Нет → не YMYL → можно облегчённый вариант

### Шаг 2. Проверь 4 блока

#### Блок E — Experience (опыт)
| Маркер | Где искать | Скоринг |
|--------|------------|---------|
| Фото автора с мест/событий | текст, alt | 0-3 |
| Личные истории («я прошёл...», «в моей практике...») | текст | 0-3 |
| Конкретные кейсы с цифрами | текст, видео | 0-3 |
| Даты, скрины, документы | текст, alt | 0-3 |
| Автор пишет от первого лица | текст | 0-3 |
| **Итого:** | | **0-15** |

**Совет для ниши Лаборатории:** добавь в каждый конспект раздел «Мой опыт применения» или «Кейс читателя» с историей.

#### Блок E — Expertise (экспертиза)
| Маркер | Где искать | Скоринг |
|--------|------------|---------|
| Имя + регалии автора | About-блок, Schema Person | 0-3 |
| Профильное образование | About-страница | 0-3 |
| Сертификаты, дипломы | About, ссылки на PDF | 0-3 |
| Ссылки на источники, исследования | текст | 0-3 |
| Терминология, точная лексика | текст | 0-3 |
| Указание методики/подхода | текст | 0-3 |
| **Итого:** | | **0-18** |

**Совет:** для конспектов — указывай «конспект подготовил: {Имя}, {регалии}». На главной книг — автора, на корпоративных — команду.

#### Блок A — Authoritativeness (авторитет)
| Маркер | Где искать | Скоринг |
|--------|------------|---------|
| Внешние ссылки с авторитетных сайтов | backlink profile (внешн.) | 0-3 |
| Упоминания в СМИ | About-страница, виджет | 0-3 |
| Отзывы клиентов/пациентов | отдельный блок, Schema Review | 0-3 |
| Публикации, выступления | About | 0-3 |
| Страница автора в соц.сетях с подписчиками | ссылки | 0-3 |
| **Итого:** | | **0-15** |

**Совет:** добавь Schema `Person` с `sameAs` (ссылки на VK, Telegram, YouTube, личный сайт).

#### Блок T — Trustworthiness (доверие)
| Маркер | Где искать | Скоринг |
|--------|------------|---------|
| SSL (https) | URL | 0-2 |
| Политика конфиденциальности | footer | 0-2 |
| Условия использования | footer | 0-2 |
| Контакты (email, телефон, адрес) | footer, страница Контакты | 0-2 |
| Юр.лицо (ООО, ИП) | footer | 0-2 |
| About-страница с командой | /about/ | 0-2 |
| Дата публикации/обновления | text, Schema | 0-2 |
| Редакционная политика | /editorial/ | 0-2 |
| Источники и сноски | text | 0-2 |
| Дисклеймеры (не заменяет консультацию) | text, footer | 0-2 |
| **Итого:** | | **0-20** |

### Шаг 3. Подсчитай общий скор

| Диапазон | Статус | Что делать |
|----------|--------|------------|
| **60-68** | Отлично | Поддерживать, обновлять |
| **40-59** | Нормально | Доработать слабые блоки |
| **20-39** | Слабо | Срочно усиливать (особенно T и E) |
| **0-19** | Критично | Страница под риском исключения из YMYL-выдачи |

### Шаг 4. Выдай рекомендации
Генерируй **конкретные действия**, не «усильте доверие», а «добавьте на страницу блок ‘Об авторе’ с фото, регалиями и ссылкой на профиль ВКонтакте».

## Выходной формат

```markdown
## E-E-A-T аудит: {URL или title}

**Тип страницы:** {YMYL / не YMYL}
**Общий скор:** {X}/68 — {статус}

### E — Experience (опыт): {X}/15
{чеклист с ✅⚠️❌ и конкретными примерами}
**Рекомендации:**
- 🔴 {срочно}
- 🟡 {важно}

### E — Expertise (экспертиза): {X}/18
{...}

### A — Authoritativeness (авторитет): {X}/15
{...}

### T — Trustworthiness (доверие): {X}/20
{...}

### Готовые блоки для внедрения

#### 1. Блок «Об авторе» (для YMYL-страниц)
```html
<div class="author-bio">
  <img src="..." alt="{Имя}, {регалии}">
  <h3>{Имя}</h3>
  <p>{регалии}</p>
  <p>{биография с маркерами опыта}</p>
  <a href="{ссылка на профиль}">Подробнее об авторе →</a>
</div>
```

#### 2. Дисклеймер (для всех YMYL-страниц)
```html
<div class="disclaimer">
  ⚠️ <strong>Важно:</strong> Эта статья носит информационный характер
  и не заменяет консультацию специалиста (психолога, врача, юриста).
  При наличии проблем со здоровьем или психикой обратитесь к профессионалу.
</div>
```

#### 3. Schema Person (для страницы автора)

#### 4. Schema Organization (для главной)

```

## Шаблоны блоков для внедрения

### Блок «Источники» (для статей)
```markdown
## Источники
1. {Автор}. *{Книга}*. — {Издательство}, {Год}. — {страницы}.
2. {Автор}. «{Название статьи}». — {Журнал}, {Год}. — {URL}.
3. Исследование: {название}. — {URL}.
```

### Блок «Дата обновления» (Schema + видимый)
```html
<p class="updated">
  Последнее обновление: <time datetime="2026-06-10">10 июня 2026</time>
</p>
```
+ в Schema `dateModified`.

### Блок «Команда» (для /about/)
```markdown
## Команда Лаборатории желаний

### {Имя} — основатель, {регалии}
- Опыт в {тема}: {N} лет
- Образование: {что}
- Публикации: {список}
- Связаться: {email}, {telegram}
```
