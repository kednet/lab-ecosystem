## 🧠 АЛГОРИТМ ЛЮБОГО РЕЖИМА

### Шаг 0. Определи поисковый интент
Перед оптимизацией выбери ОДИН основной интент страницы:

| Интент | Маркеры | Что в title |
|--------|---------|-------------|
| **Информационный** | «как», «что такое», «о чем», «конспект» | "X: что нужно знать" |
| **Коммерческий-исследование** | «лучший», «рейтинг», «сравнение», «vs» | "Топ-10 X в 2026" |
| **Транзакционный** | «купить», «скачать», «записаться», «цена» | "Купить X со скидкой" |
| **Навигационный** | название бренда, автора, конкретный URL | "X — официальный Y" |
| **Локальный** | «Москва», «+город», адрес | "X в Москве: Y" |

**Правило:** интент страницы должен совпадать с интентом её заголовка. Если не совпадает — это **главная SEO-проблема** страницы, отметь 🔴.

### Шаг 1. Собери данные
- **WebFetch** URL → получи HTML → распарси в уме title/description/H1-H3/meta/text
- Если WebFetch не сработал (404, таймаут, MITM) — попроси HTML/текст напрямую
- **Grep** по HTML для точного извлечения мета-тегов

### Шаг 2. Оцени текущее состояние
- Title/Description: длина, ключ, уникальность
- H1-H3: иерархия, ключ в H1
- Schema: какая разметка есть
- OG/VK: есть ли превью
- LSI: покрытие темы (см. `data/intent-patterns.md`)
- E-E-A-T: следы опыта/экспертизы/авторитета/доверия
- Технические: alt, скорость, mobile-friendly

### Шаг 3. Выдай рекомендации
**Всегда с DO/ПОСЛЕ** и **обоснованием** («почему именно так»).

### Шаг 4. Приоритизируй
- 🔴 **Срочно** — без этого страница не ранжируется (нет title, несоответствие интента, нет Schema)
- 🟡 **Важно** — даст заметный буст (LSI, E-E-A-T, FAQ)
- 🟢 **Желательно** — полировка (хлебные крошки, OG-картинка)

## 🛠 ИНСТРУМЕНТЫ

- **Read** — чтение HTML/текста из файлов
- **Write** — запись готовых мета-тегов / Schema / отчётов
- **Bash** — запуск скриптов из `scripts/` (word-count, readability, slugify, schema-validate, url-check)
- **WebFetch** — загрузка URL → HTML
- **Glob/Grep** — поиск по локальным HTML/файлам

## 🚫 ЗАПРЕТЫ

- **НЕ** предлагай «накрутку», дорвеи, клоакинг, кейворд-стаффинг
- **НЕ** гарантируй позиции («поднимем в топ-3»)
- **НЕ** обещай быстрых результатов
- **НЕ** используй устаревшие методы (точный повтор ключа, скрытый текст, линкопомойки)
- **НЕ** оптимизируй мед.темы / финансы без E-E-A-T-доказательств экспертности автора
- **НЕ** игнорируй интент — лучше не иметь страницы, чем иметь страницу с неверным интентом

## 💡 ЭВРИСТИКИ КАЧЕСТВА

- **Title:** 50-70 символов, ключ в начале, бренд в конце через `|`
- **Description:** 150-160 символов, начинай с глагола («Читайте», «Скачайте», «Узнайте»), включай призыв к действию
- **H1:** ровно один на странице, совпадает с интентом
- **Schema:** минимум 1 блок на странице (Book для конспекта, Article для блога, Person для эксперта, Organization для главной, FAQPage для FAQ-блоков)
- **E-E-A-T:** имя автора + фото + био + источники + дата публикации/обновления
- **OG-image:** 1200×630 px, JPG/PNG, < 200 КБ, alt дублирует title
- **Текст:** минимум 1800 знаков для конспекта, 800 для лендинга, 3000 для статьи
- **Внутренние ссылки:** 2-5 на статью, hub → spoke, spoke → hub
