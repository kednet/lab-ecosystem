# Пример: SEO-аудит сайта

## Исходные данные
```yaml
URL: https://lab.com
Тип: главная + каталог книг
Проверено: 2026-06-10
Страниц просканировано: 5 (главная, библиотека, 3 книги)
```

## Результаты аудита

### Общий скор: 47/100 — 🟡 Нормально, есть что улучшить

---

### 1. Мета-теги: 12/20

**✅ Что хорошо:**
- Title на главной есть
- Description на главной есть
- OG-теги на главной есть
- Viewport для мобильных настроен

**❌ Что плохо:**
- У 30% страниц книг нет Description
- У 60% страниц книг нет OG-тегов
- Title главной слишком длинный (78 символов, надо 50-70)
- Canonical не настроен (2 страницы могут дублироваться)

**Рекомендация:**
- 🔴 Сократить Title главной до 60-65 символов
- 🔴 Добавить Description на все страницы книг
- 🔴 Внедрить canonical повсеместно
- 🟡 Единый шаблон OG для всех страниц книг

---

### 2. Заголовки H1-H6: 8/15

**✅ Что хорошо:**
- H1 на главной есть (1 штука)

**❌ Что плохо:**
- На странице /library/ нет H2 вообще
- На 3 страницах книг нет H2
- На 2 страницах книг H1 = просто «Трансерфинг реальности» (без выгоды)

**Рекомендация:**
- 🔴 Добавить H2 на /library/ (4-6 подразделов)
- 🔴 Перестроить H1 на страницах книг: «{Название} — конспект, воркбук, отзывы»
- 🟡 H3 для подразделов внутри H2

---

### 3. Ключевые слова + LSI: 10/20

**✅ Что хорошо:**
- Главный ключ «саморазвитие» есть в Title
- Плотность ключей 2-3% (норма)

**❌ Что плохо:**
- Нет LSI-слов в текстах
- Нет alt-тегов с ключами
- Нет FAQ-блоков (PAA-вопросов)
- Ключ «исполнение желаний» не входит в H1 главной

**Рекомендация:**
- 🔴 Добавить H1 главной: «Конспекты книг по саморазвитию и исполнению желаний»
- 🟡 Внедрить LSI (см. `data/lsibase.md`) в каждый конспект
- 🟡 Добавить FAQ-блоки на каждую страницу книги
- 🟡 Alt-теги с ключами для обложек

---

### 4. Скорость и Core Web Vitals: 14/20

**✅ Что хорошо:**
- HTTPS ✅
- Mobile-friendly ✅
- LCP 1.8 сек (хорошо)
- Gzip включён

**❌ Что плохо:**
- CLS 0.15 (нужно < 0.1)
- INP 280 мс (нужно < 200)
- Изображения в JPG, не webp
- Шрифты не оптимизированы (FOIT)

**Рекомендация:**
- 🟡 Конвертировать изображения в webp/avif
- 🟡 font-display: swap для шрифтов
- 🟡 Задать размеры width/height для всех изображений (устраняет CLS)
- 🟡 Lazy loading для картинок ниже fold

---

### 5. Ссылочная структура: 9/15

**✅ Что хорошо:**
- sitemap.xml есть
- robots.txt корректен
- Внутренние ссылки есть (3-5 на страницу)

**❌ Что плохо:**
- Sitemap не в Яндекс.Вебмастере
- Sitemap не в Google Search Console
- 4 битые внутренние ссылки
- Нет BreadcrumbList schema
- Нет хлебных крошек в UI

**Рекомендация:**
- 🔴 Загрузить sitemap в Яндекс.Вебмастер и GSC
- 🔴 Починить 4 битые ссылки
- 🟡 Добавить хлебные крошки (UI + schema)
- 🟢 Добавить hreflang (если планируется мультиязычность)

---

### 6. Юзабилити + E-E-A-T: 4/10

**✅ Что хорошо:**
- SSL ✅
- About-страница есть
- Политика конфиденциальности есть
- Контакты в footer

**❌ Что плохо:**
- Нет Schema Organization
- Нет Schema Person
- Нет дат публикации/обновления
- Нет списка источников (YMYL-критично для саморазвития)
- Нет дисклеймера о «не заменяет консультацию специалиста»
- Команда «О нас» без фото и регалий

**Рекомендация:**
- 🔴 Добавить Schema Organization на главную
- 🔴 Внедрить дисклеймер на все YMYL-страницы
- 🔴 Указать дату публикации и обновления
- 🟡 Список источников на каждой странице книги
- 🟡 Фото + регалии команды на /about/
