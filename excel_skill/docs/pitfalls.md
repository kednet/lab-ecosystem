# Pitfalls — грабли Excel на Windows

## 1. cp1252 ломает `print` с кириллицей

**Симптом:**
```
UnicodeEncodeError: 'charmap' codec can't encode characters in position 2-9
```

**Причина:** стандартный вывод Windows в кодировке cp1252, а Python по умолчанию использует её для sys.stdout. Любой `print("Списание")` падает.

**Решение:** перед запуском любого скрипта:
```bash
export PYTHONIOENCODING=utf-8
```
Или однострочником:
```bash
PYTHONIOENCODING=utf-8 python scripts/excel_vlookup.py ...
```

Альтернатива — переписать скрипт на `print(..., flush=True)` с `sys.stdout.reconfigure(encoding='utf-8')`, но env-переменная проще.

## 2. `PermissionError: [Errno 13]` — файл открыт в Excel

**Симптом:**
```
PermissionError: [Errno 13] Permission denied: 'мой.xlsx'
```

**Причина:** Windows блокирует файл, пока он открыт в Excel (даже просто для просмотра).

**Решение:** закрыть файл в Excel и запустить скрипт снова. Все скрипты скила ловят эту ошибку и печатают понятное сообщение, но открыть файл сами не могут.

## 3. Артикулы как текст с ведущими нулями

**Симптом:** артикул `00789` в Excel отображается как `789`. После `openpyxl.load_workbook` ключи `00789` и `789` могут «сравниться» как одно и то же число, и маппинг в vlookup сломается.

**Решение:** включать `--key-as-text` в `excel_vlookup.py` — скрипт делает `str(key).strip()` и сравнивает как строки.

## 4. Числа с запятой vs с точкой

**Симптом:** в одной колонке `163,847`, в другой `163.847`, итого разные числа.

**Причина:** Excel хранит числа внутренне как `float` (с точкой), но отображает по локали. openpyxl всегда читает `float` с точкой. Если источник отдал строку `"163,847"` (например, из выгрузки 1С), будет `string`, и сравнение с `float` молча провалится.

**Решение:** предварительно привести колонку к числу в Excel (`Данные → Текст по столбцам → заменить , на .`), или добавить нормализацию в скрипт.

## 5. Корпоративный TLS-MITM

В этом стеке openpyxl **не лезет в сеть**, поэтому `*_VERIFY_SSL=false` не нужен. Но если какой-то скрипт скила вдруг полезет (например, `excel_to_csv.py --upload`), прокси и verify-SSL нужно выставить по памяти `[[corporate-mitm-proxy]]`.

## 6. Формулы vs значения

`excel_sum_column.py` пишет **формулу** `=G2*H2`. В готовом файле в Excel всё пересчитается. Но если открыть в LibreOffice / Numbers / Google Sheets — пересчёт может отличаться (округление, формат даты).

Чтобы «заморозить» значения (числа, не формулы):
```python
import openpyxl
wb = openpyxl.load_workbook("file.xlsx", data_only=False)  # формулы как строки
ws = wb["Списание"]
for r in range(2, ws.max_row + 1):
    cell = ws.cell(row=r, column=9)
    if isinstance(cell.value, str) and cell.value.startswith("="):
        cell.value = None  # очистится при следующем открытии
wb.save(...)
```
Но проще: после `excel_sum_column.py` открыть в Excel, `Ctrl+;` (или скопировать-вставить как значения) и сохранить.

## 7. `.xlsm` с VBA-макросами

openpyxl **читает структуру** (листы, ячейки, формулы), но **не редактирует VBA-код**. Если в `.xlsm` нужны макросы — пользуйтесь `xlwings` (только Windows + Excel) или `pywin32`.

## 8. Очень большие файлы

Файлы >50 МБ или >100К строк лучше читать через `pandas`:
```python
import pandas as pd
df = pd.read_excel("огромный.xlsx", sheet_name="Данные", usecols=["A", "F", "H"], dtype={"A": str})
```
openpyxl грузит весь файл в память и на 100К строк начинает тормозить.

## 9. Скрытые листы / защищённые книги

openpyxl по умолчанию видит скрытые листы. Защищённые паролем книги (`Защитить книгу`) openpyxl **откроет** (пароль на изменение), но если файл зашифрован целиком — нужна `msoffcrypto-tool`.

## 10. `data_only=True` после записи формул

`excel_sum_column.py` записывает формулы через обычный `load_workbook` (не `data_only`). Когда Excel потом откроет и сохранит, кэш значений обновится. Если хочется проверить значения **сразу** после записи (без открытия в Excel) — это не сработает: openpyxl не вычисляет формулы. Используйте `pycel` или `formulas` (вне скила).
