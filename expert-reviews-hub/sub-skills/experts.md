# Подскил: EXPERTS (эксперты)

## Назначение
Сбор, структурирование и публикация данных об экспертах (психологи, коучи, авторы, преподаватели) для экосистемы Лаборатории желаний. Включает:
- Карточку эксперта (Person/Schema)
- Связь с обработанными книгами в WL
- Поиск упоминаний в СМИ и интернете
- Каталог ресурсов эксперта

## Режимы

### 1. `/expert card {Имя}` — карточка эксперта

#### Вход
- Имя эксперта (обязательно)
- Контекст (опционально): тема, ниша, страна

#### Алгоритм
1. **WebSearch** — найти официальный сайт, биографию, регалии
2. **WebFetch** — загрузить официальный сайт → спарсить bio
3. **Grep по WL** — найти упоминания эксперта в `output/*/summary.md`
4. **Сгенерировать карточку** по шаблону `templates/expert-card.md`
5. **Готовая Schema Person** по `templates/expert-schema.json`

#### Выход
```yaml
# experts/{slug}.json
name: "Вадим Зеланд"
slug: "vadim-zeland"
role: "Автор метода Трансерфинг реальности"
photo: "https://..."
short_bio: "..."
long_bio: "..."
credentials: ["...", "..."]
specializations: ["трансерфинг", "саморазвитие"]
contacts:
  website: "..."
  vk: "..."
  telegram: "..."
  email: "..."  # только публичный
works_for: "..."
books_in_wl: ["Трансерфинг реальности", "Шелест утренних звёзд"]
mentions_count: 47
```

#### Файлы
- `experts/{slug}.json` — структурированные данные
- `experts/{slug}.md` — человекочитаемая карточка
- `experts/{slug}.schema.json` — JSON-LD Person
- `experts/{slug}.report.md` — отчёт со всеми источниками

#### См. промпт
`prompts/expert-card.md` — детальный алгоритм

### 2. `/expert mentions {Имя}` — упоминания в интернете

#### Вход
- Имя эксперта

#### Алгоритм
1. **WebSearch** — найти упоминания в СМИ, подкастах, на других сайтах
2. **Фильтр** — отделить реальные упоминания от SEO-спама
3. **Категоризация**:
   - 📰 Пресса (статьи, интервью)
   - 🎙 Подкасты (эпизоды, цитаты)
   - 📺 YouTube (видео, лекции)
   - 📚 Цитирование (упоминания в книгах)
   - 💬 Обсуждения (форумы, соцсети)
4. **Скоринг** — по DA домена, дате, объёму
5. **Bundle** — топ-20 упоминаний

#### Выход
```markdown
# Упоминания: {Имя}

## 📰 Пресса (5)
- 2026-05-12 — "Как Зеланд изменил..." — ria.ru
- ...

## 🎙 Подкасты (3)
- 2026-04 — "Трансерфинг и наука" — podcast.ru

## 📺 YouTube (12)
- 2026-06-01 — "Вадим Зеланд: интервью" — 1.2M просмотров
```

#### Файлы
- `mentions/{slug}.json` — структурированный список
- `mentions/{slug}.md` — читаемый bundle

#### См. промпт
`prompts/expert-mentions.md`

### 3. `/expert resources {Имя}` — каталог ресурсов эксперта

#### Вход
- Имя эксперта

#### Что ищем
- 📚 **Книги** (ISBN, год, издательство, обложка)
- 🎓 **Курсы** (платформы: Skillbox, Нетология, Udemy, свои)
- 🎥 **Видео** (YouTube, VK Video, RuTube)
- 🎙 **Подкасты** (как ведущий или гость)
- 📝 **Статьи** (СМИ, блоги, научные журналы)
- 🌐 **Сайты/проекты** (официальный, школа, метод)

#### Алгоритм
1. **WebSearch** по запросам:
   - `"{Имя}" книга купить`
   - `"{Имя}" курс`
   - `"{Имя}" youtube`
   - `"{Имя}" официальный сайт`
2. **WebFetch** — спарсить каждую найденную страницу
3. **Структурировать** по типу ресурса
4. **Для книг** — найти ISBN, обложку, цену (через Литрес/Ozon)
5. **Для видео** — embed URL, длительность, просмотры

#### Выход
```yaml
# resources/{slug}.json
books:
  - title: "Трансерфинг реальности"
    year: 2004
    isbn: "978-5-9573-2563-4"
    cover: "https://..."
    buy_links: ["litres.ru/...", "ozon.ru/..."]
courses:
  - platform: "Skillbox"
    title: "..."
    url: "..."
    price: "..."
videos:
  - platform: "YouTube"
    title: "..."
    url: "..."
    views: 1.2M
    duration: "45:20"
```

#### Файлы
- `resources/{slug}.json` — структурированный каталог
- `resources/{slug}.md` — читаемый каталог

#### См. промпт
`prompts/expert-resources.md`

### 4. `/expert link {Имя}` — связь с книгами в WL

#### Вход
- Имя эксперта
- (опционально) путь к `output/` WL — по умолчанию `C:\Users\kfigh\wish_librarian\output\`

#### Алгоритм
1. **Grep** — найти все упоминания имени в `output/*/summary.md` и `output/*/workbook.md`
2. **Для каждого совпадения:**
   - Извлечь контекст (абзац ± 200 символов)
   - Определить роль: автор, цитирование, рекомендация, критика
   - Сохранить цитату дословно
3. **Сгруппировать по книгам**
4. **Подсчитать статистику**:
   - Сколько книг связано
   - Сколько цитат
   - Топ-3 темы (по частотности)

#### Выход
```markdown
# Связи с WL: {Имя}

## Обработанные книги ({N} шт)

### Трансерфинг реальности (Вадим Зеланд) — АВТОР
- Упоминаний: 47
- Цитат: 12
- Роли: автор
- Темы: визуализация, маятник, пространство вариантов

### [другая книга] — ЦИТИРОВАНИЕ
- Упоминаний: 3
- Цитата: "..."
```

#### Файлы
- `wl-links/{slug}.json` — структурированные связи
- `wl-links/{slug}.md` — читаемый отчёт

#### См. промпт
`prompts/expert-wl-link.md`

### 5. `/expert find {тема}` — поиск экспертов по теме

#### Вход
- Тема (например, «когнитивно-поведенческая терапия»)
- (опционально) критерии: страна, язык, платформа

#### Алгоритм
1. **WebSearch** — `{тема} эксперт`, `{тема} психолог`, `{тема} автор`
2. **WebFetch** — профили на b17.ru, psysovet.ru, kp.ru (для РФ)
3. **Скоринг** — по регалиям, цитируемости, активности
4. **Топ-10** экспертов с кратким описанием

#### Выход
```markdown
# Топ-10 экспертов: {тема}

1. **Имя Фамилия** — {регалии}
   - 50K подписчиков VK, 12 публикаций
   - Сайт: ...
   - Контакт: ...

2. ...
```

## 🔗 ИНТЕГРАЦИЯ С SEO ADVISOR

Карточка эксперта экспортирует:
- `experts/{slug}.schema.json` → готов к вставке на страницу книги или лендинг
- `experts/{slug}.md` → блок «Об эксперте» в карточке
- Schema → добавляется к существующему `schema.json` книги

## 🔗 ИНТЕГРАЦИЯ С WISHLIBRARIAN

При обработке книги:
1. Парсер извлекает `book.author`
2. Скил `/expert card` создаёт/обновляет карточку автора
3. Добавляет в `book.metadata.json` поле `expert_slug`
4. WL при следующей обработке этой книги подтянет карточку

## 📊 СТАТИСТИКА

После создания карточки — добавь в `experts/index.json`:
