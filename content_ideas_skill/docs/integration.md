# Интеграции с другими скилами

## Карта интеграций

```
                    ┌──────────────────────────┐
                    │  content-ideas-skill     │
                    │  (наш скил)              │
                    └──────────────────────────┘
                          ▲          │
              читает      │          │  отдаёт
                          │          ▼
        ┌─────────┬───────┴──┬─────────────┬─────────────┐
        │         │          │             │             │
   ┌────▼───┐ ┌───▼────┐ ┌──▼─────┐ ┌─────▼──────┐ ┌────▼─────┐
   │WishLib-│ │Wish-   │ │Expert  │ │SEO Advisor │ │Publisher │
   │rarian  │ │Coach   │ │&Reviews│ │            │ │          │
   │        │ │        │ │Hub     │ │            │ │          │
   └───┬────┘ └───┬────┘ └────────┘ └──────┬─────┘ └─────┬────┘
       │          │                         │             │
       │ источник │ источник               │ оптимизирует│ публикует
       │ тем      │ тем                    │ готовый     │ готовые
       │          │                        │ текст       │ посты
       └──────────┘                        └─────────────┘
```

## 1. WishLibrarian → content-ideas-skill (← чтение)

**Что тянем:**
- список книг (`wish_librarian/books/`)
- по каждой книге: цитаты, ключевые идеи, темы
- готовые конспекты (если есть) — для блог-статей

**Формат передачи:**
- WL хранит книги в markdown/yaml → `extract_themes.py --source wl` парсит
- результат: список тем с цитатами в формате, пригодном для prompt

**Когда:**
- раз в неделю при планировании контента
- по запросу `--source wl --book-id <id>`

**Зависимость:** низкая. Если WL недоступен, работаем с захардкоженным списком книг.

## 2. WishCoach → content-ideas-skill (← чтение)

**Что тянем:**
- модули детектора (5 модулей × 20 вопросов)
- типы желаний (4 тона × 5 интенсивностей)
- воркбуки (для мини-уроков)

**Формат передачи:**
- Coach хранит модули в JSON/markdown → `extract_themes.py --source coach` парсит
- результат: темы «как отличить навязанное от истинного» в разных форматах

**Когда:**
- каждую неделю — минимум 1 пост на тему детектора (уникальное позиционирование)

**Зависимость:** низкая. Можно работать без Coach, если есть PRD v2.0 в `C:\Users\kfigh\coach_agent\`.

## 3. Expert & Reviews Hub → content-ideas-skill (← чтение)

**Что тянем:**
- готовые обзоры книг (expert-режим)
- рецензии (review-режим)
- hub-статистику (какие книги заходят аудитории)

**Формат передачи:**
- адаптируем обзор в короткий пост (рерайт угла + цитата)
- используем как источник идей «что вышло в нише»

**Когда:**
- когда выходит новая книга / обзор — генерируем 2–3 поста на его основе

**Зависимость:** средняя. Expert & Reviews Hub может быть не активен — работаем без него.

## 4. content-ideas-skill → SEO Advisor (→ передача)

**Что отдаём:**
- идеи для блога с черновыми ключами
- структуру статьи (H1/H2) на основе идеи

**Что получаем:**
- оптимизированный список ключей и LSI
- рекомендации по объёму, плотности
- FAQ-схему
- meta-теги, OG

**Формат передачи:**
- `data/ideas-bank.json` с пометкой `--target blog` → SEO Advisor берёт
- результат: оптимизированная идея-карточка

**Когда:**
- перед рендером блог-статьи

**Зависимость:** средняя. Если SEO Advisor не активен, делаем руками.

## 5. content-ideas-skill → Publisher (→ передача)

**Что отдаём:**
- готовые карточки идей (`templates/post-idea.md`)
- контент-план на период
- привязку к типу (VK/блог/TG) и дате

**Что получаем:**
- готовый пост (текст + визуал)
- публикацию в VK / на сайт
- архив

**Формат передачи:**
- `export_publisher.py --ideas data/ideas-bank.json --target vk` создаёт файлы в формате Publisher
- Publisher берёт их в работу по своему пайплайну

**Когда:**
- по контент-плану (за день/неделю до публикации)

**Зависимость:** средняя. Publisher — основной канал; если его нет, экспортируем в markdown.

## Полный цикл поста

```
1. content-ideas-skill: генерирует идею
   ↓
2. (опц.) SEO Advisor: оптимизирует под блог
   ↓
3. Publisher: рендерит текст (LLM)
   ↓
4. Publisher: добавляет визуал
   ↓
5. Publisher: публикует + анонсирует
   ↓
6. (опц.) Expert & Reviews Hub: собирает метрики
   ↓
7. content-ideas-skill: обновляет history.json (дедуп)
```

## Принципы интеграции

1. **Слабая связанность** — наш скил не падает, если другой недоступен
2. **Одно направление** — мы отдаём карточки идей, а не тексты
3. **Локальные данные** — интеграция через файлы, не API (для простоты)
4. **Версионирование форматов** — карточка идей имеет version, чтобы старые версии не ломали новые
