# Специфика блога на сайте

## Контекст

Блог на сайте «Лаборатории желаний» — это **полноценные статьи** для:
- SEO-трафика (Яндекс + Google)
- Лид-магнита для подписчиков app.pulab.online
- Глубокого раскрытия тем, которые в VK не помещаются
- Навигации по книгам WL и модулям Coach

См. PRD в `C:\Users\kfigh\coach_agent\PRD.md` (для связанного проекта).

## Технические особенности формата блога

### Длина
- **Минимум:** 1500 знаков (для SEO)
- **Оптимум:** 2500–4000 знаков
- **Максимум:** 6000 (дальше — сложно удержать внимание, дробить)
- **Для SEO-статей:** 3000–5000 знаков

### Структура статьи
```markdown
# H1 — главный заголовок (1 штука, ключ в начале)

Краткий лид (1-2 предложения, что получит читатель)

## H2 — первый раздел

Текст + пример

## H2 — второй раздел

Список или подразделы H3

## H2 — «Что с этим делать» / Практика

Пошаговая инструкция

## H2 — Заключение

+ CTA (подписка, воркбук, книга)
```

### Что работает в блоге
- ✅ **H1/H2/H3 структура** — и для SEO, и для читаемости
- ✅ **Списки и подпункты** — легче сканировать
- ✅ **Длинные абзацы допустимы** — здесь читают внимательнее, чем в VK
- ✅ **Ссылки на WL/Coach** — навигация по экосистеме
- ✅ **FAQ-блок в конце** (→ передать в SEO Advisor для schema.org)
- ✅ **Мета-описание** (description) — обязательно
- ✅ **Изображения с alt-текстами** — для SEO
- ✅ **Внутренние ссылки** на другие статьи блога

### Что НЕ работает в блоге
- ❌ **Слишком короткие статьи** (менее 1000 знаков) — Google считает thin content
- ❌ **Вода и вступления «в этой статье мы поговорим о…»** — сразу к делу
- ❌ **Стоковые фото** — лучше обложки-иллюстрации в стиле ЛЖ
- ❌ **«Кликай сюда» в тексте** — плохо для SEO
- ❌ **Только текст без визуала** — скучно

## SEO-требования (базовые)

| Элемент | Требование |
|---|---|
| Title (title тег) | 50–60 символов, ключ в начале |
| Meta description | 150–160 символов, ключ + CTA |
| H1 | 1 штука, уникальный |
| H2 | 2–5 штук, содержат LSI-ключи |
| URL | короткий, на латинице, с ключом (slug) |
| Alt у картинок | описательный, с ключом (если уместно) |
| Внутренние ссылки | минимум 2 на статью |
| Внешние ссылки | на авторитетные источники (1–3) |

**Детали:** `seo-advisor-skill` (если активен).

## Темы для блога (отличия от VK)

В блоге лучше раскрываются:
- **Глубокие разборы книг WL** (2000+ знаков)
- **Методологии детектора Coach** (5 модулей × детальный разбор)
- **Подборки книг** (для SEO-запросов «книги про …»)
- **Практики и воркбуки** (для лид-магнита)
- **Ответы на частые вопросы** (FAQ → SEO-трафик)

## Цикл жизни статьи

1. **Идея** (наш скил) → карточка `templates/blog-idea.md`
2. **Спека** (SEO Advisor) → ключи, LSI, FAQ-схема
3. **Рендер** (Publisher) → готовый текст + визуал
4. **Публикация** (Publisher) → на сайт
5. **Анонс в VK** → пост-тизер со ссылкой в комментах
6. **Обновление** (раз в 3–6 мес) → актуализировать данные, добавить новые ссылки

## Связь с SEO Advisor

Когда скил SEO Advisor активен:
- Идея блога → передаётся в SEO Advisor
- SEO Advisor возвращает: ключи, LSI, FAQ, структуру H1/H2, meta-теги
- Publisher рендерит с учётом SEO-пакета

Когда SEO Advisor не активен:
- Идея блога → ручная SEO-проработка (title, description, ключи)
- Скелет статьи → рендер в Publisher

| Критерий | VK-пост | Блог-статья |

| Длина | 800–1500 | 2500–4000 |
| Цель | вовлечение | трафик + лид-магнит |
| SEO-упор | минимальный | высокий |
| Ссылка | редко | часто |
| CTA | коммент | подписка / скачать |
| Тон | короче, живее | подробнее, академичнее |
