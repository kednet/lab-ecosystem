# IT-phonetics — как произносятся IT-термины

## Проблема

IT-термины часто:
- Похожи на обычные слова, но произносятся иначе (cache, data, async)
- Латинские/греческие корни (architecture, asynchronous, database)
- Аббревиатуры (HTTP, API, SQL, JSON, CI/CD)

## Фонетические правила

### -ch в конце

- **tech** = тек (не "тех")
- **cache** = кеш
- **search** = сёрч
- **approach** = эп-роуч
- **架构** = нет, не пишем по-китайски :)

### Финальный -e

- **code** = коуд
- **type** = тайп
- **database** = дейта-бейс
- **architecture** = ар-ки-тек-чер

### Греческие/латинские корни

- **async** = эй-синк (a- как "эй", не "а")
- **asynchronous** = эй-син-кро-нес
- **synchronous** = син-кро-нес
- **asymmetric** = эй-си-мет-рик

### Согласные clusters

- **nth** = энс (как "эн-тый" в "ninety")
- **SQL** = си-квел (не "с-ку-эль" по буквам)
- **JSON** = джей-сан (не "джей-сон")

## Как практиковать

1. **TTS** (Phase 2) — послушать правильное произношение
2. **Forvo.com** — живые произношения от носителей
3. **Google Translate** — наведи на слово → иконка динамика
4. **YouGlish.com** — произношение в контексте YouTube-видео
5. **Зеркало** — произнеси вслух 5 раз, проверь что язык/губы в правильной позиции
