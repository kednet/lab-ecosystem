## 🌐 Telegram-бот: прокси для РФ

`api.telegram.org` блокируется на уровне провайдера. Решение — пустить бота
через SOCKS5/HTTP-прокси. WishLibrarian поддерживает оба протокола.

### 1.1. Установите/поднимите прокси

Самые популярные варианты в РФ:

| Сервис / тип | Что это | Где взять |
|--------------|---------|-----------|
| **Outline** | Shadowsocks-сервер за 5 минут | <https://getoutline.org/> |
| **Amnezia** | OpenVPN/WireGuard с обходом DPI | <https://amnezia.org/> |
| **V2Ray/Xray** | VLESS/VMess/Trojan | Поднять на VPS |
| **X-UI / 3X-UI** | Web-панель для V2Ray | GitHub |
| **Корпоративный** | HTTP/SOCKS на работе | У сисадмина |
| **Свой VPS** | shadowsocks-libev | DigitalOcean, Aéza, и т.д. |

Для теста подойдёт и публичный (с оговорками про безопасность).

### 1.2. Получите URL прокси

Форматы, которые понимает WishLibrarian:

```
socks5://user:pass@host:1080
socks5://host:1080                ← без авторизации
socks4://user@host:1080
http://user:pass@host:8080
https://host:443
127.0.0.1:1080                   ← автоматически трактуется как socks5
```

### 1.3. Укажите в `.env`

```env
TELEGRAM_BOT_TOKEN=8837346894:AAFJtHPwzE62--fwBl2wJ9A2jPTGq8SG8hc
TELEGRAM_PROXY_URL=socks5://user:pass@127.0.0.1:1080
```

### 1.4. Проверьте

```bash
# Запустите и смотрите логи
python -m agent.telegram_bot

# Должно появиться:
# 🌐 Telegram через прокси: socks5://***
# 🤖 WishLibrarian Bot is running. Press Ctrl+C to stop.
```

Откройте в Telegram своего бота → `/start` → должны получить приветствие.

### 1.5. Проксирование HTTP-запросов (не только Telegram)

Если прокси нужно и для других HTTP-вызовов (Yandex/Claude), установите ещё:

```env
# Через aiohttp (использует Telegram, VK-бот)
TELEGRAM_PROXY_URL=socks5://...
VK_PROXY_URL=socks5://...

# Через requests (используют парсеры)
HTTP_PROXY=socks5://user:pass@host:1080
HTTPS_PROXY=socks5://user:pass@host:1080
```

## 👥 ВКонтакте-бот: создание сообщества

### 2.1. Создайте сообщество

1. Откройте <https://vk.com/groups> → **Создать сообщество**
2. Выберите тип: **Публичная страница** (если книжный клуб) или **Группа** (если закрытый клуб)
3. Название: например, `Книжный клуб WishLibrarian`
4. Тематика: «Литература», «Образование» или «Хобби»

### 2.2. Включите сообщения сообщества

1. Зайдите в сообщество → **Управление** (справа сверху) → **Настройки**
2. Левое меню → **Сообщения**
3. Включите: **Сообщения сообщества** → **Включены**
4. Внизу нажмите **Сохранить**

> ⚠️ Пока сообщения выключены — бот не сможет никому писать.

### 2.3. Включите Long Poll API

1. **Управление** → **Настройки** → **Работа с API**
2. Вкладка **Long Poll API**:
   - **Long Poll API**: включено
   - **Версия API**: 5.131 или новее
3. Вкладка **Типы событий** — оставьте всё включённым по умолчанию
4. **Сохранить**

### 2.4. Получите токен

1. **Управление** → **Настройки** → **Работа с API**
2. Вкладка **Ключи доступа** → **Создать ключ**
3. Поставьте галочки:
   - ✅ **Разрешить отправку сообщений**
   - ✅ **Доступ к сообщениям сообщества**
   - ✅ (опционально) другие, если нужны
4. Подтвердите через 2FA (если включена)
5. Скопируйте токен — он начинается с `vk1.a.…`

### 2.5. Узнайте ID сообщества

Это **числовой** ID (не путать с коротким адресом `vk.com/club123`).

Способы:
1. **В URL сообщества**: `vk.com/club987654321` → ID = `987654321`
2. **В настройках**: Управление → Настройки → Адресная строка → ID
3. **Через API** (когда есть токен):
   ```bash
   curl https://api.vk.com/method/groups.getById?access_token=ВАШ_ТОКЕН&v=5.131
   ```

### 2.6. Заполните `.env`

```env
VK_GROUP_TOKEN=vk1.a....длинная_строка....
VK_GROUP_ID=123456789
VK_PROXY_URL=                        # пусто — VK API доступен из РФ
```

### 2.7. Запустите

```bash
python -m agent.vk_bot
```

Ожидаемый вывод:
```
✅ Подключен к сообществу: Книжный клуб WishLibrarian (id=123456789)
🤖 WishLibrarian VK Bot is running. Группа: Книжный клуб WishLibrarian (id=123456789)
Нажмите Ctrl+C для остановки.
```

### 2.8. Тест

1. Откройте ваше сообщество в ВК
2. Нажмите **Написать сообщение** (или перейдите в раздел сообщений сообщества)
3. Отправьте `/start` — бот должен ответить приветствием
4. Попробуйте `/doctor` — диагностика

## 🤖 Команды ВК-бота

| Команда | Что делает | Пример |
|---------|-----------|--------|
| `/start`, `/help` | Справка | `/start` |
| `/add <URL>` | Обработать книгу | `/add https://www.koob.ru/zeland/level1` |
| `/list [N]` | Последние N книг | `/list 20` |
| `/search <запрос>` | Поиск по библиотеке | `/search привычки` |
| `/book <название>` | Прислать summary.md | `/book трансерфинг` |
| `/export <fmt> <название>` | Экспорт в txt/html/pdf | `/export txt трансерфинг` |
| `/doctor` | Диагностика | `/doctor` |

Свободный URL без `/add` тоже обрабатывается.

### Отличия от Telegram-бота

| Возможность | Telegram | ВКонтакте |
|-------------|----------|-----------|
| Inline-кнопки | ✅ | ❌ (нет callback в личных сообщениях) |
| Длинные сообщения | разбивка на части | разбивка на части |
| Файлы | документ | документ (через `doc` attachment) |
| Markdown | ✅ HTML | ⚠️ BBCode + `<b>`/`<i>` (мы используем `<b>`) |
| Админ-фильтр | через `TELEGRAM_ADMIN_ID` | отсутствует (любой может писать) |
| Прокси | SOCKS5/HTTP | обычно не нужен (api.vk.com доступен) |

## 📦 Формат файлов и прокси

### Что читает бот из `.env`

| Переменная | Где | Описание |
|------------|-----|----------|
| `TELEGRAM_BOT_TOKEN` | оба | Токен от @BotFather |
| `TELEGRAM_PROXY_URL` | только Telegram | SOCKS5/HTTP для api.telegram.org |
| `TELEGRAM_ADMIN_ID` | только Telegram | int, 0 = принимать всех |
| `VK_GROUP_TOKEN` | только ВК | Токен сообщества |
| `VK_GROUP_ID` | только ВК | int, ID сообщества |
| `VK_PROXY_URL` | только ВК | Обычно не нужен |

### Приоритет чтения токенов

1. `settings.telegram_bot_token` (из `.env`)
2. → fallback → `os.environ["TELEGRAM_BOT_TOKEN"]`
3. → fallback → ошибка `sys.exit(1)`

## 🧯 Типичные ошибки

### `RuntimeError: no running event loop` (Telegram-бот)

В нашей версии исправлено: `_build_proxy_connector` теперь сам создаёт loop
если его нет. Если всё равно видите — обновите `agent/telegram_bot.py`.

### `❌ VK_GROUP_TOKEN не задан`

`VK_GROUP_TOKEN` пуст в `.env`. Заполните по инструкции выше.

### `❌ Токен невалиден или сообщество недоступно`

Токен скопирован с пробелами / потерян символ / сообщество удалено. Проверьте:

```bash
curl "https://api.vk.com/method/groups.getById?access_token=ВАШ_ТОКЕН&v=5.131"
```

Если ответ — `{"error": {...}}` — токен невалиден.

### VK: «Сообщения сообщества выключены»

Управление → Настройки → Сообщения → **Включены** → Сохранить.

### VK: «Permission denied» при отправке сообщения

В ключе доступа не включена опция **«Разрешить отправку сообщений»**.
Создайте новый ключ с правильными галочками.

### Telegram: «SOCKS proxy error»

Прокси-сервер не отвечает / требует другой схемы. Проверьте:
- хост:порт доступны (`telnet host port`)
- логин/пароль верные
- протокол именно `socks5://` (не `https://`)

### Telegram: бот не видит сообщения

Long Poll / polling требует **ровно один процесс** с этим токеном. Если бот
запущен в двух местах — он будет конфликтовать.

### ВК: бот видит сообщения, но не отвечает

Проверьте `event.to_me` в коде. В Long Poll фильтр `event.to_me` означает
«только сообщения, отправленные в ЛС сообщества». Если пользователь пишет
в комментариях — нужен другой фильтр.

## 🚀 Запуск обоих ботов одновременно

```bash
# Терминал 1
python -m agent.telegram_bot

# Терминал 2
python -m agent.vk_bot
```

Они используют **разные** токены (Telegram и VK) — конфликтов нет.

Через systemd (Linux-сервер):

```ini
# /etc/systemd/system/wish-tg.service
[Service]
ExecStart=/opt/wl/.venv/bin/python -m agent.telegram_bot
EnvironmentFile=/opt/wl/.env

# /etc/systemd/system/wish-vk.service
[Service]
ExecStart=/opt/wl/.venv/bin/python -m agent.vk_bot
EnvironmentFile=/opt/wl/.env
```

```bash
sudo systemctl enable --now wish-tg wish-vk
```

Через Docker — два отдельных контейнера с одним `.env`-файлом через `-v`.
