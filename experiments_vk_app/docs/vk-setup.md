# Привязка URL к VK Mini App (app_id 54643519)

Пошаговая инструкция для администратора — после того, как файл мини-аппа задеплоен и отдаёт HTTP 200.

## 1. Убедиться, что мини-апп задеплоен

```bash
curl -I https://app.pulab.ru/experiments-app/
```

Ожидаемый ответ:
```
HTTP/2 200
server: nginx
content-type: text/html
```

Если 404 — `dist/experiments-app/index.html` ещё не залит (вернись к `README.md`, секция «Деплой»).

## 2. Открыть кабинет приложения

Перейти: **https://vk.com/editapp?id=54643519**

Это настройки именно того приложения «Поделиться экспериментом», что виден в сообществе «ЛАБОРАТОРИЯ ЖЕЛАНИЙ» (id=237295798).

В левом меню — вкладки: **Информация**, **Размещение**, **Оформление**, **Отображение**, **Правовые документы**, **Другие площадки**, **Дополнительные**, **Разработка**, **Модерация**, **Фичеринг**.

## 3. Вкладка «Размещение»

Перейти на вкладку **«Размещение»** в левом меню.

В форме должно быть одно или несколько полей:
- **«Адрес приложения»** (или «URL», «Ссылка на приложение»)
- (опционально) **«Домен»** — если VK разделяет домен и путь

Прописать:
- **Адрес**: `https://app.pulab.ru/experiments-app/`
- **Домен** (если есть): `app.pulab.ru`

Нажать **«Сохранить»** (или «Применить»).

## 4. Дождаться модерации (если требуется)

Если после сохранения появится плашка «На модерации» — VK проверит приложение в течение нескольких часов/дней (обычно 1-3 дня для нового приложения). Пока идёт модерация, мини-апп может не открываться через `https://vk.com/app54643519`, но будет доступен по прямой ссылке в кабинете (есть кнопка «Открыть» или «Предпросмотр»).

## 5. Проверить в мобильном ВК

Открыть в мобильном приложении ВКонтакте ссылку:
```
https://vk.com/app54643519
```

Или через само сообщество «ЛАБОРАТОРИЯ ЖЕЛАНИЙ» (https://vk.com/club237295798) → правый блок «Приложения» → Поделиться экспериментом → **«Запустить»**.

Должна открыться наша форма: интро с кнопкой «Поделиться», 5 шагов на одном экране, финальный экран с подтверждением и id истории.

## 6. Что проверить внутри мини-аппа

Внутри ВК-клиента должны работать:
- **VKWebAppInit** (без ошибок в DevTools) — если открыт «Предпросмотр», в консоли видно `vk-bridge: initialized`
- **Тактильная обратная связь** — на телефоне при нажатии «Дальше» должна быть лёгкая вибрация
- **Шаринг «ВКонтакте»** — открывает нативный диалог ВК (не web-окно)
- **POST на API** — после 5 шагов получить id истории в формате `#20260619-xxxxxx`

В web-версии (`https://app.pulab.ru/experiments-app/` в обычном браузере) всё то же, но без вибрации и шаринг идёт через `vk.com/share.php` (fallback).

## 7. Если что-то не работает

| Симптом | Что делать |
|---|---|
| 404 / белый экран в ВК | Проверить `curl -I https://app.pulab.ru/experiments-app/` — должно быть 200. Проверить, что в кабинете VK в «Размещении» указан именно `https://app.pulab.ru/experiments-app/` (с трейлинг-слешем и `https://`) |
| «Нарушение политики» / блок | VK модерирует после первой публикации. Подождать 1-3 дня, обычно без последствий для белого приложения |
| `ReferenceError: vkBridge is not defined` | VK Bridge не загрузился. Проверить, доступен ли `https://unpkg.com/@vkontakte/vk-bridge/dist/browser.min.js` (в РФ может быть заблокирован). Внутри ВК скрипт инжектится автоматически. Если ошибка в обычном браузере — это нормально, код обернут в `if (window.vkBridge)` |
| Мини-апп открывается, но кнопки не реагируют | Открыть «Предпросмотр» с `?dev=1` или через DevTools ВК — посмотреть console errors |
| POST возвращает `got_too_short` | Поле «Что получилось» короче 30 знаков. В UI это проверяется до отправки, но если что-то проскочило — увеличить текст |
| POST возвращает 5xx | Проверить `curl https://api.pulab.online/api/experiments` — endpoint должен быть жив. Логи: `journalctl -u lab-api -f` на VPS |
| Другое | Написать в поддержку Лаборатории |

## 8. Когда всё работает

- Проверить, что в `wish_librarian/.env` есть запись про новый mini app (опц., если потом понадобится)
- Можно публиковать в сообщество пост с ссылкой `https://vk.com/app54643519` — кнопка «Запустить» появится в карточке приложения
- Идти итерировать: история уже сохраняется в общую БД experiments (та же, что и TG-бот, и web-форма), так что в `/experiments/` подборке новая запись появится автоматически после одобрения модератором
