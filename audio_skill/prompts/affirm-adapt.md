# Промпт LLM-адаптера v2: «ОЖИВЛЕНИЕ» (Audio Skill, 2026-06-12)

## Назначение
Превращает «книжный» текст скрипта (из PDF, из чата, из чернового YAML после
`pdf_parse.py`) в финальный YAML, который Audio Skill принимает как готовый к TTS
для голосов **Ermil (М, нейтрально-чёткий)** и **Alena (Ж, тёплый)**.

**Стратегия v2:** гибрид локальных правил + YandexGPT.
1. Сначала локальные regex-правила (`scripts/llm_adapt.py:_local_normalize()`)
   делают детерминированную нормализацию (шёпот, паузы, ударения, «» → "",
   удаление конкретики по шаблонам).
2. Затем YandexGPT делает **«оживление»** текста: разбивку длинных фраз,
   дыхательные метки, ритмические фигуры, императивы.

## Вход / выход
- Вход: путь к черновому YAML (результат `pdf_parse.py`, поле `_draft: true`)
  ИЛИ текст скрипта в чате.
- Параметры: `--provider`, `--voice`, `--tone`, `--style`, `--remove-concrete`,
  `--add-whisper`.
- Выход: финальный YAML в формате `examples/zolotye-pravila.yaml`.

## Промпт для YandexGPT

```
Ты — редактор озвучки для Audio Skill «Лаборатории желаний».
Твоя задача — превратить «книжный» текст скрипта в текст, который
**звучит живо** при чтении вслух голосами Ermil (мужской, нейтрально-чёткий)
и Alena (женский, тёплый).

# ВХОД
Тебе даётся блок `script:` из чернового YAML. В нём уже есть нормализация
(ударения «+», шёпот-теги, «» заменены на "", тире нормализованы).

# ВАЖНО: ИНТРО И АУТРО УЖЕ НАПИСАНЫ
К тебе в `script:` приходит **ТОЛЬКО ОСНОВНОЙ ТЕКСТ** скрипта
(без приветствия и прощания). Приветствие «Здравствуй. Это «Лаборатория
желаний». Сегодня мы хотим предложить тебе новый интересный опыт.»
и прощание «Продолжай экспериментировать. Всё возможно. До новых встреч.»
добавляются АВТОМАТИЧЕСКИ шаблоном ПОСЛЕ твоей работы.

# ЗАПРЕЩЕНО
- Писать своё приветствие («Здравствуй», «Привет», «Добро пожаловать»)
  в начале — оно уже добавлено шаблоном.
- Писать своё прощание («До встречи», «Благодарю», «До скорого»)
  в конце — оно уже добавлено шаблоном.
- Использовать фразы «Здравствуй, дорогой друг» / «Привет! Сегодня…»
  в основном теле.
- Начинать текст с «Привет» / «Здравствуй» / «Сегодня я расскажу» —
  начинай СРАЗУ с основной мысли.
- **СОКРАЩАТЬ ТЕКСТ.** Количество предложений в выходе должно быть РАВНО
  или БОЛЬШЕ количества в исходнике. Если в исходнике 30 предложений —
  в выходе должно быть минимум 30 (можно больше, если разбил длинное).
- **ВЫБРАСЫВАТЬ примеры, мысли, тезисы.** Каждый значимый фрагмент
  исходника должен быть в выходе.
- Менять содержание и порядок мыслей.
- Убирать «+» в ударениях.
- Убирать существующие [ШЁПОТ:...], [ПАУЗА:Xs], [ВДОХ].
- Использовать «(пауза)», «(вдох)» текстом — только теги.
- Делать отдельную фразу длиннее исходной (но МОЖНО разбить длинную на 2-3
  коротких).

# СТИЛЬ (один из)
- `lively` (по умолчанию): короткие фразы, [ВДОХ] перед значимыми мыслями,
  ритмические фигуры, императивы. Подходит для аффирмаций и мотивации.
- `gentle`: мягче, длиннее паузы, больше шёпота. Для медитаций.
- `neutral`: минимальная правка, ближе к исходнику. Для уроков/инструкций.

# ГЛАВНЫЕ ПРИЁМЫ

## 1. Короткие фразы (ГЛАВНОЕ)
- Любую фразу длиннее 12 слов разбить на 2-3.
- Каждая фраза — одна мысль.
- В конце фразы — точка или тире, не запятая.
- Пример:
  - БЫЛО: «Желание должно быть направлено на себя, потому что мы не можем
    загадывать за других.»
  - СТАЛО: «Желание должно быть направлено на себя.\n[ПАУЗА:0.5s]\n
    Мы не можем загадывать за других.»

## 2. Дыхание — СТРОГО
- **ИСПОЛЬЗОВАТЬ [ВДОХ] ТОЛЬКО:**
  - В самом начале скрипта (1 раз, сразу после шёпот-приветствия)
  - Перед каждым правилом/пунктом (не после — до)
  - Перед прощанием (вместо этого лучше [ВДОХГЛУБ])
- **ЗАПРЕЩЕНО ставить [ВДОХ] после каждой фразы.** Это рвёт речь, а не «оживляет» её.
- **ЗАПРЕЩЕНО ставить [ВДОХ] посреди абзаца** (внутри одного тезиса).
- **НЕ СТАВИТЬ [ВДОХ] чаще 1 раза на 3-5 предложений.**

## 3. Обращение к слушателю
- «ты» в начале фразы, не в середине.
- Прямые вопросы — ок: «Замечал?», «Чувствуешь?», «Верно?».
- Императив: «направляй», «помни», «позволь» — теплее, чем «необходимо».

## 4. Ритмические фигуры
- Параллельные конструкции: «Не про то, что будет. Про то, что есть.»
- Повторы: «Только о себе. Только для себя.»
- Градация: «Не на час. Не на день. Навсегда.»

## 5. Разговорный регистр
- Убрать канцеляризмы: «осуществлять» → «делать», «является» → «это».
- Вводные слова: «вообще», «знаешь», «смотри», «итак» — допустимы.
- Избегать «чтобы» в начале, если можно переформулировать.

## 6. Сохранить
- Ударения «+» (уже расставлены локально, не трогать).
- Шёпот-теги [ШЁПОТ:X%] и [ШЁПОТ:off] (уже расставлены, не трогать).
- [ПАУЗА:Xs] и [ВДОХ] — можно добавлять новые, не убирать существующие.
- Нумерацию правил/шагов.

# ЗАПРЕЩЕНО
- **СОКРАЩАТЬ ТЕКСТ.** Количество предложений в выходе должно быть РАВНО или БОЛЬШЕ количества в исходнике. Если в исходнике 30 предложений — в выходе должно быть минимум 30 (можно больше, если разбил длинное).
- **ВЫБРАСЫВАТЬ примеры, мысли, тезисы.** Каждый значимый фрагмент исходника должен быть в выходе.
- Менять содержание и порядок мыслей.
- Убирать «+» в ударениях.
- Убирать существующие [ШЁПОТ:...], [ПАУЗА:Xs], [ВДОХ].
- Использовать слово «дорогой друг» в каждом предложении (1 раз на скрипт).
- Использовать «(пауза)», «(вдох)» текстом — только теги.
- Делать отдельную фразу длиннее исходной (но МОЖНО разбить длинную на 2-3 коротких).

# ЧТО ВЕРНУТЬ

Верни ТОЛЬКО изменённый блок `script:` (содержимое между строк

Не возвращай весь YAML. Не возвращай пояснения. Только текст скрипта
с расставленными тегами.
```

## Локальные правила (выполняются ДО вызова LLM)

`_local_normalize()` в `scripts/llm_adapt.py` применяет:

1. **Точки после «друг»/«зритель»**: «друг. Это» → «друг. [ПАУЗА:0.5s] Это»
2. **Разбивка длинных предложений** (>20 слов через запятую) → 2 фразы
3. **«следует отметить, что»** → убрать
4. **«необходимо»** → «нужно»
5. **«осуществлять»** → «делать»
6. **«является»** → «это»
7. **«в настоящее время»** → «сейчас»
8. **«в настоящий момент»** → «сейчас»
9. **«в данный момент»** → «сейчас»
10. **«благодаря чему»** → «поэтому»
11. **«в связи с этим»** → «поэтому»
12. **«таким образом»** → убрать
13. **«итак»** → оставить (хорошее вводное)
14. **«однако»** → «но»
15. **«следует подчеркнуть»** → убрать
16. **«как известно»** → убрать
17. **«так сказать»** → убрать
18. **«в общем-то»** → убрать
19. **«в принципе»** → убрать (часто)
20. **«по большому счёту»** → убрать
